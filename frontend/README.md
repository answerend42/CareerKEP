# Career KG 前端

这是 `Career KG` 的前端工作台，只在 `frontend/` 目录内维护。

## 目标

- 输入自然语言画像和结构化证据。
- 调整信心、探索、负向容忍度等演示参数。
- 查看图谱传播快照。
- 解释正式推荐、near miss、桥接建议和目标岗位缺口。
- 运行极端输入鲁棒性测试，检查噪声、冲突、稀疏和中英混合输入下的识别稳定性。

## 技术栈

- TypeScript
- Vite

## 本地运行

先进入 `frontend/` 目录安装依赖，再启动开发服务。

```bash
npm install
npm run dev
```

默认会启动在 `http://127.0.0.1:5173`。

## 构建

```bash
npm run build
```

## 测试

```bash
npm test
```

这个命令会先执行前端构建，再运行 `scripts/robustness-check.ts`，用正常场景和极端场景做一轮烟测。

## 后端联调

前端默认会把 `/api/recommend` 代理到 `http://127.0.0.1:8000`。

如果后端没有启动，页面会自动回退到本地模拟推理结果，方便单独演示。

## 演示参数说明

- `信心权重`：放大直接命中的证据和方向分数。
- `探索权重`：增加 bridge 候选和传播过程里的探索感。
- `负向容忍度`：降低 blocker 对结果的压制，适合展示“放宽筛选”后的变化。

## 鲁棒性测试

前端内置了几组极端输入场景，放在“极端测试”按钮里：

- `噪声输入`：长文本、重复词和无关片段混合。
- `冲突输入`：同一段话同时包含正向和否定信号。
- `稀疏输入`：几乎没有明确技能证据。
- `中英混合`：英文缩写、符号和中文混写。

这些场景会直接刷新右侧的鲁棒性测试结果，帮助观察解析、归一和权重是否需要继续调整。

右侧的鲁棒性测试面板会跟随当前调参一起刷新，所以你在左侧改了权重以后，可以直接对比同一批极端输入的结果变化。

面板里还会显示“默认权重”对比值和变化百分比，方便判断某次调参是在改善鲁棒性，还是把某些场景拉偏了。

面板顶部还会单独标出本轮调参里提升最多和回落最多的极端场景，便于快速定位要继续加固的输入模式。

鲁棒性面板还会给出下一步调参建议，通常会优先提示探索权重、信心权重或否定词/抑制逻辑是否需要继续调整。
